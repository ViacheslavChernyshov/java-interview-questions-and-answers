# GitHub SEO Setup

This file contains ready-to-use GitHub repository settings for stronger SEO, better click-through rate, and cleaner project positioning.

## 1. Recommended Repository Name

Primary option:

`java-interview-questions-and-answers`

Shorter alternatives:

- `java-interview-questions`
- `java-developer-interview-questions`
- `java-backend-interview-questions`

Recommended choice:

- Use `java-interview-questions-and-answers` if available
- Keep the current repository owner profile consistent with Java / backend / interview positioning

## 2. GitHub About Description

Primary description:

`500+ Java interview questions and answers for Junior, Middle, and Senior developers. Covers Spring Boot, Hibernate, SQL, Kafka, Docker, Kubernetes, and more.`

Alternative shorter version:

`500+ Java interview questions and answers covering Spring Boot, Hibernate, SQL, Kafka, Docker, Kubernetes, Collections, Concurrency, and more.`

## 3. Website URL

Use after GitHub Pages is published:

`https://<github-username>.github.io/java-interview-questions-and-answers/`

If a custom domain is added later, replace the GitHub Pages URL with the custom domain in the repository settings.

## 5. Social Preview Concept

Main headline:

`500+ Java Interview Questions`

Secondary line:

`Junior • Middle • Senior`

Topic strip:

`Spring Boot • Hibernate • SQL • Kafka • Docker • Kubernetes`

Visual direction:

- clean light background
- strong dark text
- 3-level badge row for Junior / Middle / Senior
- compact topic chips
- avoid noisy gradients and decorative clutter

## 6. Repository Settings Checklist

- Set repository name to the recommended SEO-friendly slug
- Add the About description
- Add the website URL after Pages is live
- Add all relevant topics
- Upload a custom social preview image
- Pin the repository on the GitHub profile if this project is strategic

## 7. Positioning Notes

- English should remain the primary SEO entry point
- Russian can stay as a strong alternate language version
- Ukrainian should be added to search-facing navigation only after real content is published
- GitHub repository SEO helps discovery, but GitHub Pages should become the main indexable surface for Google
